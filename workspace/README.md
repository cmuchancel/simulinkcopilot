# Workspace Layout

`workspace/` is the non-code area of the repo.

- `benchmark/`
  Canonical benchmark systems and manifests.
- `data/`
  Derived datasets and benchmark catalogs.
- `docs/`
  Walkthroughs, narrative artifacts, and demo documentation.
- `examples/`
  Bundled example LaTeX systems.
- `generated_models/`
  Generated Simulink models and related build outputs.
- `bedillion_demo/`
  Demo-specific outputs and verbose walkthrough bundles.
- `reports/`
  Generated benchmark reports, GUI saved runs, debug traces, and regression artifacts.
- `paper/`
  Manuscript and paper-related assets.

Implementation code should not live under `workspace/`.
